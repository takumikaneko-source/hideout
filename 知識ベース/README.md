# 知識ベース

作業を横断して使い回せる知見を、テーマ別フォルダで管理する。
「その作業だけの記録」は作業履歴へ、「次も使える知識」はここへ。

## フォルダ構成

```
知識ベース/
├── aws/          # AWSサービスの挙動・制約・ハマりどころ
├── sql/          # SQLパターン・調査クエリ集
├── python/       # Python実装パターン・ライブラリの知見
├── 組織・環境/   # 社内制約（プロキシ・証明書・権限まわり等）
├── 運用/         # 運用手順・障害対応の教訓
└── その他/       # 上記に収まらないもの（増えたらフォルダ昇格）
```

- フォルダは必要になったときに増やす（空フォルダを先に作らない）
- 1テーマのファイルが増えてきたら、サブフォルダに整理する

## 記事の書き方

- `テンプレート/ナレッジ記事.md` を使用
- 1ファイル = 1トピック。ファイル名は内容がわかる日本語（例: `TLS検査プロキシ対応.md`）
- 「結論 → 背景 → 詳細 → 出典（どの作業で得たか）」の順で書く
- 出典として `作業履歴/YYYYMMDD_*/` へのリンクを必ず残す
