# hideout — クロスプロジェクト知識庫

システムエンジニアの作業実績・参考資料・ナレッジを一元管理するリポジトリ。
taxi-cloud をはじめ、複数プロジェクトの設計・実装・運用知識を横断的に集約する。

- リモート: https://github.com/takumikaneko-source/hideout.git
- デフォルトブランチ: master
- Claude Code 向けの指示は `CLAUDE.md`（ルート直下）を参照

## ディレクトリ構成

```
hideout/
├── 作業履歴/        # プロジェクト実施履歴（日付フォルダ YYYYMMDD_名称）
├── 参考資料/        # 設計資料・仕様書・参照資料
├── テンプレート/    # 共通資産・テンプレート・アイコン
├── 知識ベース/      # 横断的ナレッジ（SQL・パターン集など）
├── CLAUDE.md        # Claude Code 向けプロジェクト指示（自動読込）
└── README.md
```

## 用途

- taxi-cloud および他プロジェクトの設計・実装・運用知識を集約
- 参照資料・手順書・設計説明の保管
- チーム間・プロジェクト間の知識共有
- 過去実績の検索・再利用

## 作業の進め方

1. 新規作業・プロジェクト開始時に `作業履歴/YYYYMMDD_名称/` フォルダを作成する
2. 設計・実装・テスト結果を該当フォルダに格納し、`README.md` に作業内容・背景・結果を記録する
3. 完了時に commit・push する

### 資料の配置ルール

| 種類 | 配置先 |
|------|--------|
| 参照資料・設計書・仕様 | `参考資料/` |
| テンプレート・共通部品・アイコン | `テンプレート/` |
| 実装履歴・成果物・試行錯誤記録 | `作業履歴/YYYYMMDD_*/` |
| 共有ナレッジ・教訓・パターン集 | `知識ベース/` |

## Git 管理

- コミットメッセージは日本語で記述する
- 機能単位の変更は `【プロジェクト】内容` 形式を推奨（例: `【taxi-cloud】CloudWatchログ収集ツール配布版作成`）
- 資料の大きな更新・新規ナレッジ追加時に commit を切る

## 注意事項

- **機密情報は含めない**（AWS認証情報・パスワード・秘密鍵・個人認証情報は `.gitignore` で継続的に除外）
- 個人名・役職名などの特定情報は避け、「運用チーム」等で一般化する
- Google Sheets（.gsheet）ファイルはバイナリのため、ローカルでは参照扱いになる
- 日本語のみ使用する
