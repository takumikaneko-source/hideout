アラート判定ツール 使い方ガイド

2026-07-16 ／ 運用担当者向け・操作手順

クイックスタート

このガイドは AI判定ツール（②）の使い方です。入力となる「エラーログ転記済みのxlsx」は、収集ツールで用意済みのものを使います。

  1. 準備：セットアップ.bat → claude を導入・ログイン → 環境チェック.bat で「すべて良好」を確認
  2. 入力を置く：ログ転記済みの xlsx を 1_ログ収集ツール/OUTPUT/ に置く（既定の入力先）
  3. 判定：2_判定ツール/判定実行.bat をダブルクリック（まずは MAX_ROWS を小さく）
  4. 確認：2_判定ツール/OUTPUT/ の判定済みxlsxを見て、E列に最終判断を記入
重要 AIの判定は参考情報です。最終判断は必ず運用チームの目視（E列）で行ってください。

1. これは何か

収集済みのエラーログ（Excelの D列）を AI が読み、対応要否（要 / 静観 / 不要）と影響度（大 / 中 / 小）を判定して、判定済みExcelの G〜K列に書き込むツールです。ログの収集は別のツール（収集ツール）で行い、その出力を入力として使います。

1-1. フォルダの見方

作業者が触るのは、判定ツール直下の 判定実行.bat・settings.py・ナレッジ / OUTPUT だけです。本体プログラムは program/ にまとめてあり、開く必要はありません。

場所役割
環境チェック.bat / セットアップ.bat準備用（点検・依存導入）
2_判定ツール/判定実行.bat判定を実行（★これをダブルクリック）
2_判定ツール/settings.py設定（ここだけ編集）
2_判定ツール/ナレッジ/判定基準・過去事例（正本はGoogleドライブ／後述）
2_判定ツール/OUTPUT/判定結果・差分記録が出る
2_判定ツール/program/本体・依存・説明（触らない）
資料/全体仕様・このガイド

2. 準備（初回のみ）

  1. Python を用意：python.org 版を推奨。注意 Microsoft Store 版は不具合が出ることがあります（後述）。
  2. 依存パッケージを導入：セットアップ.bat をダブルクリック。
  3. Claude CLI を導入してログイン： 
    npm install -g @anthropic-ai/claude-code
claude
    APIキーは不要です。社内プロキシで SELF_SIGNED_CERT_IN_CHAIN が出たら「6. 困ったとき」を参照。
  4. 点検：環境チェック.bat を実行し「すべて良好」を確認。足りない物があれば対処が表示されます。
安全 環境チェック.bat は AI も AWS も呼ばず、資格情報も読みません。何度実行しても安全です。

3. 判定の使い方

  1. 入力を用意：エラーログ転記済みの xlsx を 1_ログ収集ツール/OUTPUT/ に置く（settings.py の INPUT_XLSX が既定でここを指します）。別の場所にあるなら INPUT_XLSX をそのパスに変更。
  2. 少数で試す：2_判定ツール/settings.py の MAX_ROWS を 5 など小さくする。
  3. 実行：判定実行.bat をダブルクリック。

処理対象は「D列にログがあり、G列が空」の行です。結果は 2_判定ツール/OUTPUT/…_判定済.xlsx の G〜K列に出ます。

枠の消費に注意 既定は案B（Opusオーケストレーション・サブスク経由）で、1行につきAIを複数回呼びます。件数を増やすときは使用枠と時間に注意してください（目安: 数行で数分・少額）。

4. 結果の見方

列内容
G: AI対応要否要 / 静観 / 不要
H: AI影響度大 / 中 / 小
I: AI判定理由【結論】【根拠】【確認事項】【前後関係】の4段落
J: AI確信度高 / 中 / 低（材料不足なら低）
K: 判定方式どのモデル・経路で判定したか

理由（I列）は次のように、ブロックごとに空行・1文ごとに改行して読みやすく整形されます。

【結論】ログ欠落のため断定できず静観とします。

【根拠】過去5事例は対応不要でした。
本件はログが無く両立します。

【確認事項】同エラーが連続発報していないか確認してください。

【前後関係】読取APIのため下流波及はなし。

5. 最終判断と差分の記録

  • 判定済みファイルの E列（対応要不要） に、運用チームが最終判断を記入します（ツールはE列を上書きしません）。
  • AI判定と運用判断の差分は、判定ツールが自動で OUTPUT/AI判定_運用差分記録.xlsx に蓄積します（判定済みファイルは毎回作り直されるため、差分は別ファイルに残します）。
  • 差分記録の「運用対応要否」を入れると「差分」列が自動で 一致／差分 を表示します。理由メモを溜めて、ナレッジ（判定基準・事例）の改善に使います。

6. 困ったとき

まず 環境チェック.bat を実行すると、多くを自動検出して対処を表示します。

症状原因対処
claude … が見つかりませんでしたClaude CLI 未導入npm install -g @anthropic-ai/claude-code → claude でログイン
npm で SELF_SIGNED_CERT_IN_CHAIN社内TLS検査プロキシ下の「社内証明書」参照
claude.cmd を「なし」と誤検出Microsoft Store 版 Python で AppData が見えないpython.org 版 / pyenv 版（実体）を使う。判定実行.bat は実体を自動選択します
パッケージ未導入エラー依存が未インストールセットアップ.bat を実行
保存／書き込みに失敗対象 Excel を開いたままその Excel を閉じて再実行
使用上限 で未判定サブスクの使用枠上限時間をおいて再実行。MAX_ROWS を小さく

6-1. 社内証明書（プロキシでSSLエラー）

Windowsの信頼済み証明書を1ファイルに書き出し、npm に渡します（PowerShellで実行）。

$out = "$env:USERPROFILE\corp-ca-bundle.pem"
$certs = Get-ChildItem Cert:\LocalMachine\Root, Cert:\CurrentUser\Root
$lines = foreach ($c in $certs) { "-----BEGIN CERTIFICATE-----"; [Convert]::ToBase64String($c.RawData,'InsertLineBreaks'); "-----END CERTIFICATE-----" }
$lines | Set-Content -Encoding ascii $out
npm config set cafile "$out"

7. ナレッジの正本（Google ドライブ）

判定に使うナレッジ（rules.yaml / cases.jsonl / guidelines.md / system_context.md）の正本は Google ドライブ共有フォルダです。最新はそこで管理します。判定ツールはオフラインで動くため、各PCではローカルに用意した最新版を settings.py の KNOWLEDGE_DIR で参照させます。

  • (A) 同期して参照（推奨）：Google ドライブ デスクトップで共有フォルダを同期し、その同期パスを KNOWLEDGE_DIR に指定。初期設定 Google サインインと共有フォルダの追加という作業者の操作が必要です（社内承認が要る場合あり）。
  • (B) 手動ダウンロード：共有フォルダから4ファイルを 2_判定ツール/ナレッジ/ に上書き。

同梱の ナレッジ/ は初期スナップショット（フォールバック）です。詳しい場所は はじめにお読みください.md に記載しています。

8. 上級者向け：ターミナルから直接実行

実体の Python が入ったターミナル（PowerShell 等）に慣れている方は、.bat を使わず直接実行してもかまいません。

cd 2_判定ツール
python program\judge_logs.py --help   # 使い方の確認
python program\judge_logs.py          # 判定を実行
注意 その場合は「実体のPython」であること（Store版でないこと）を自分で担保してください。判定実行.bat は、実体Pythonの自動選択・UTF-8設定・画面保持を代わりに行っています。
