# ログ収集ツール（運用チーム向け）

Slack通知アラート収集スプレッドシートの未処理行について、CloudWatch のログを取得し、
入力ファイルのコピーに転記するツールです。ログの判定にAIは使いません。

## 同梱ファイル

| ファイル | 説明 |
|---|---|
| `collect_logs.py` | ツール本体（**編集不要**） |
| `settings.py` | 設定ファイル（**★ここだけ編集**） |
| `requirements.txt` | 必要なパッケージ |
| `手順書.html` | 詳しい手順（ブラウザで開く） |

## 使い方（3ステップ）

**1. 準備（初回だけ）**

```
pip install -r requirements.txt
```

**2. 設定**

`settings.py` を開き、少なくとも次の2つを自分の環境に合わせて書き換える。

- `INPUT_XLSX` … ダウンロードした .xlsx のパス
- `PREFIX_TO_PROFILE` … アラーム接頭辞 → AWSプロファイル名 の対応

**3. 実行**

```
python collect_logs.py
```

途中で MFA コードの入力を求められます。終わると `（入力名）_転記済.xlsx` ができます。

## 結果の貼り付け

出力ファイルの **D列にログ本文**、**F列にリンク** が入ります。
本物のスプレッドシートの「**同じ行の D列・F列だけ**」を貼り付けてください。

> ⚠️ ファイルを丸ごとアップロード（差し替え）しないこと。ダウンロード〜貼り付けの間にも
> 自動収集が新しい行を追記しているため、丸ごと差し替えると追記分が消えます。

## D列の状態（結果の見方）

| D列の中身 | 意味 | 対応 |
|---|---|---|
| ログ本文が入っている | 取得成功 | D/F を貼り付け |
| `【該当ログなし】` | その時間帯にログなし | 必要なら `WINDOW_MIN` を広げて再取得 |
| `【取得できず】` | 未対応の種類・アラーム未検出・日時不正 | 理由を確認し手動対応 |
| 空のまま | 一時的なエラー（AWS通信・認証） | **もう一度実行**すると再取得 |

原因を直して取り直したいときは、その行の **D列を空にして** 実行すると再取得します。

## 事前に必要なもの（AWS）

- `~/.aws/config` にスイッチロール用プロファイル（`role_arn` / `source_profile` / `mfa_serial`）
- 対象ロールの権限:
  `cloudwatch:DescribeAlarms` / `logs:DescribeMetricFilters` /
  `logs:FilterLogEvents` / `logs:GetLogEvents`
  （Step Functions のアラームも扱う場合は `states:ListExecutions` / `states:GetExecutionHistory` も必要）

## 費用

AWSからのログ取得費用は、現状の件数では**実質ほぼ0円**です
（課金対象の Logs Insights を使わない設計）。詳細は `手順書.html` の「費用」を参照。

## 困ったとき

- **証明書エラー（SSL）** … `settings.py` の `CA_BUNDLE` に社内CA(.pem)のパスを指定。
- **MFAを複数回聞かれる** … アカウントごとに1回。使い回せないので毎回新しいコードを入力。
- **「入力xlsxが見つかりません」** … `settings.py` の `INPUT_XLSX` のパスを確認。

詳しくは `手順書.html` を参照してください。
