GAS Web App 接続テスト 手順書

「ローカルPythonから、サーバー上のGASを起動できるか」を、あなたの手元で検証する

所要: 10分 Q5（公開デプロイ可否）の確認も兼ねる 関連: gas_test_Code.gs / gas_test_client.py
目的 0. 事前準備 1. GASを用意 2. Web Appデプロイ 3. Python実行 4. 結果の見方 補足

目的

本方式の肝である「ローカルのPython → サーバー上のGAS」の起動（往復通信）が成立するかを確認する。

このテストは GoogleもAWSも認証しません（HTTPでGAS Web Appを叩くだけ）。
同時に Q5＝「アクセス=全員」でWeb Appを公開デプロイできるか も判明します（組織ポリシー確認を兼ねる）。
推奨: 本番のPhase1プロジェクトではなく、テスト用に新規スプレッドシート＋スクリプトで試す（本番のデプロイに影響しないため）。

0事前準備

pip install requests

1テスト用GASを用意

  1. 新規スプレッドシートを作成（https://sheets.new）
  2. メニュー 拡張機能 → Apps Script
  3. 既定の Code.gs の中身を全消去し、gas_test_Code.gs の内容を丸ごと貼り付けて保存
  4. コード上部の SHARED_TOKEN を任意の合言葉に変更（例: taxi-test-2026）

2ウェブアプリとしてデプロイ

  1. 右上 デプロイ → 新しいデプロイ
  2. 種類の選択（歯車）→ ウェブアプリ
  3. 設定:
    • 説明: 任意
    • 実行するユーザー: 自分
    • アクセスできるユーザー: 全員
  4. デプロイ → 初回は アクセスを承認（自分のアカウントで許可）
  5. 表示される ウェブアプリ URL（末尾が /exec） をコピー
ここで判明するQ5: 「アクセスできるユーザー」で 「全員」が選べない／デプロイがブロックされる 場合は、組織が公開Web Appを制限しています。その旨をメモしてください（→ サービスアカウント等の代替検討へ）。

3Pythonから実行

gas_test_client.py の上部を編集:

WEBAPP_URL   = "（コピーした /exec URL）"
SHARED_TOKEN = "（GASと同じ合言葉）"
SHEET_ID     = ""   # 任意: シート読取も試すなら対象シートのID

実行:

python gas_test_client.py

4結果の見方

応答意味判定
"ok": true ＋ "message": "hello from GAS"ローカル→サーバーGASの起動成功この方式で進められる
（SHEET_ID指定時）spreadsheetName / a1 が返るGASからのシート読取も成功読取もOK
"ok": false, "error":"invalid token"合言葉の不一致GASとPythonの SHARED_TOKEN を一致させる
JSONでなくHTMLが返るURL違い／未デプロイ／アクセス制限URL末尾 /exec・デプロイ済み・アクセス=全員 を確認
「全員」が選べずデプロイ不可組織が公開Web Appを制限Q5: 代替方式を検討

補足

  • GETが成功すれば「起動できる」ことの証明として十分です。POSTはApps Scriptのリダイレクト仕様でボディが落ちる場合がありますが、ok:true が返れば起動自体は成功しています（本実装では送り方を調整します）。
  • このテストで作ったスプレッドシート／デプロイは、確認後に削除して構いません。
  • 結果は「ok:true が出た」か「全員でデプロイできずブロック」かだけ共有いただければ十分です（URLやトークンの共有は不要）。
