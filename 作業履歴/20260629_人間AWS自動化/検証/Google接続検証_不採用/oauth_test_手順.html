OAuthユーザー認証 接続テスト 手順書

「自分（ログインユーザー）として OAuth 認証し、スプレッドシートを読み書きできるか」を手元で検証する

所要: 15分 2つの関門: ①OAuth許可（組織ポリシー） ②シート読み書き 関連: oauth_test.py
目的 0. 事前準備 1. プロジェクト/API 2. OAuthクライアント発行 3. テストシートID 4. 実行 5. 結果の見方

目的

サービスアカウントが使えない場合の代替「OAuthユーザー認証」が、この組織で成立するかを確認する。関門は2つ：

  1. OAuthクライアントを発行でき、同意画面で「許可」できるか（組織が自作アプリのOAuth接続を許すか）
  2. そのまま自分の権限でスプレッドシートを読み書きできるか
このテストは Googleのみ を扱い、AWSの資格情報は使いません。実行・認証はすべてあなたの環境で完結します（結果は「成功／どこで詰まったか」だけ共有すれば十分）。
SA方式との違い: OAuthはあなた自身として動くため、シートを誰かに共有する操作は不要（あなたが見えるシートならそのまま読み書き可）。一方、使う人ごとに初回のブラウザ認証が要ります。

0事前準備

pip install gspread google-auth google-auth-oauthlib truststore

truststore は社内SSL検査(プロキシ)対策です（OSの証明書ストアを使用）。

1プロジェクトとAPI有効化

  1. Google Cloud Console に会社アカウントでアクセス
  2. プロジェクトを新規作成（例: oauth-test）。作成できない場合は組織制限の可能性 → メモして報告
  3. APIとサービス → ライブラリ で Google Sheets API と Google Drive API を有効化

2OAuthクライアントを発行 ★関門①

  1. APIとサービス → OAuth同意画面:
    • User Type は 内部(Internal) を選択（組織内のみで使う）→ アプリ名・サポートメール等を入力して保存
    • 「内部」が選べず「外部」しか無い場合は、外部で作成し「テストユーザー」に自分のメールを追加
  2. APIとサービス → 認証情報 → 認証情報を作成 → OAuthクライアントID:
    • アプリの種類: デスクトップアプリ → 作成
    • 表示されたダイアログで JSONをダウンロード
  3. ダウンロードしたJSONを credentials.json にリネームし、oauth_test.py と同じフォルダに置く
★関門①の判定: 後の実行(手順4)でブラウザに 「アクセスをブロックしました」「管理者の承認が必要」 等が出て先へ進めない場合、組織が自作アプリのOAuth接続を制限しています＝OAuth方式はNG。その旨をメモして報告してください（→ ④AWS中継 等の検討へ）。

3テストシートのIDを用意

スプレッドシートを開いたときのURL https://docs.google.com/spreadsheets/d/★ここがID★/edit の ★ 部分を控える。

  • 書き込みまで試す（推奨）: 本番を汚さないため、あなたが見える場所に空のスプレッドシートを新規作成（例 oauth_test_dummy）し、そのIDを使う。
  • OAuthは「あなた自身」として動くので、共有設定は不要（あなたが開けるシートならアクセス可）。

4スクリプト設定 & 実行

CREDENTIALS_FILE = "credentials.json"
SHEET_ID         = "（控えたID）"
ALLOW_WRITE      = True
CA_BUNDLE        = ""   # 社内CA(.pem)があればパス。無ければ truststore が使われる
python oauth_test.py
  • 初回はブラウザが開く → 会社アカウントでログイン → 「許可」
  • 2回目以降は token.json が使われ、ブラウザは開きません。

5結果の見方

表示意味判定/対処
[1] -> 認証OK組織のOAuthアプリ接続は許可されている関門①クリア
[2] 読み取り成功 ＋ [3] 書き込み結果: oauth_write_okOAuthでの読み書き成功OAuth方式で進める
ブラウザで「アクセスをブロック」「管理者の承認が必要」組織がOAuthアプリを制限OAuth方式NG → 代替へ
シートを開けません … PermissionError(403)シートにアクセス権がない / API未有効自分が開けるシートのIDか／Sheets API有効化を確認
SSL: CERTIFICATE_VERIFY_FAILED社内プロキシのCA未信頼pip install truststore、または CA_BUNDLE に社内CA(.pem)を指定
このテストで分かること: ①組織がOAuthアプリ接続を許すか ②自分の権限でシート読み書きできるか。両方OKなら、OAuth方式（各自が初回だけブラウザ認証）で進められる。
