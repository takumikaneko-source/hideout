# 作業履歴

プロジェクト・タスク単位の実施記録を日付フォルダで管理する。

## 命名規則

```
YYYYMMDD_作業名/
```

- 日付は**着手日**（例: `20260709_hideout初期化`）
- 作業名は内容がわかる簡潔な日本語
- 長期にわたる作業も着手日のフォルダで継続する（日付を作り直さない）

## フォルダ内の標準構成

```
YYYYMMDD_作業名/
├── README.md      # 作業記録（必須。テンプレート/作業記録.md を使用）
├── 資料/          # 設計資料・説明資料（必要時）
├── 成果物/        # スクリプト・SQL・設定ファイル等（必要時）
└── メモ/          # 試行錯誤・一時メモ（必要時）
```

小さい作業は README.md 1枚＋成果物ファイル直置きでよい。フォルダを義務にしない。

## 記録のタイミング

- 着手時: フォルダと README.md を作成（背景・目的だけ先に書く）
- 作業中: 判明したこと・決定事項を README.md に追記
- 完了時: 結果・残課題を記入し commit

## 知識ベースへの昇格

作業の中で得た「他の作業でも使い回せる知見」（SQLパターン、AWS の挙動、組織の制約など）は、
作業履歴に埋めたままにせず **知識ベース/ に別記事として切り出す**。
作業履歴側の README.md からはリンクで参照する。
